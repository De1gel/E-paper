## 2. 已做过的实验类型
- 伪局刷（部分行重写）实验

- 黑白网格 + 随机块更新实验

## 3. 关键观察
- “看起来局部更新”与“真正稳定局刷”不是一回事。
- 当渲染路径混入全刷初始化时，会掩盖局刷效果。
- 将局刷逻辑从主流程中剥离后，工程可维护性明显提高。

## 4. 当前决策
- 主流程已回归产品路径，不默认跑局刷实验。
- 局刷保留为独立模块能力，按需调用。
- 测试说明移动到 `test/partial_refresh/README.md`。

## 5. 后续实验建议
1. 固定背景 + 固定小块重复刷新（10/100/500 次）记录残影。
2. 小块位置分布测试：中心、边缘、角落。
3. 颜色组合测试：黑白、白红、白黄、白橙近似。
4. 对比全刷恢复效果：每 N 次局刷后执行一次全刷。
