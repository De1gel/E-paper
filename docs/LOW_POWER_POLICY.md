# 低功耗策略（项目强制约束）

更新时间：2026-04-01  
依据：`D:\MyCode\ESP\设计思路资料\低功耗思路.txt`

## 1. 选型结论
本项目默认采用**方案 3（最低功耗）**：
- 刷新完成后先让墨水屏进入 `deep sleep`
- 再通过板级供电控制脚（`IO32`）切断外设供电
- 目标是接近“显示保持、系统静态功耗极低”的状态

说明：
- 方案1（仅 deep sleep）和方案2（待机）保留为兼容兜底，不作为默认策略。

## 2. 板级实现约束
- `IO32 = HIGH`：外设上电（渲染前）
- `IO32 = LOW`：外设断电（渲染后）
- 每次重新上电后，必须执行 SPI/EPD 初始化流程，不允许跳过。

## 3. 软件时序约束
1. 渲染前：`IO32 HIGH -> SPI init -> EPD init`
2. 渲染后：`EPD_sleep() -> IO32 LOW`
3. 非交互时段：ESP32 进入 `light/deep sleep`（按模式分级）

## 4. 模式分级功耗策略
- `Normal`：允许休眠；轮播间隔内 MCU 不应空转。
- `ConfigWait`：允许短时活跃，但超时（60s）必须退出并回低功耗路径。
- `ConfigAP/STA`：网络会提高功耗，需在功能结束后自动回 `Normal` 并触发低功耗收敛。

## 5. 开发时禁止项
- 禁止在 `loop()` 中长期 busy wait。
- 禁止未刷新时保持 EPD 供电。
- 禁止新增功能绕过模式超时机制。

## 6. 验收指标（阶段性）
- 屏幕刷新后，外设供电必须被关闭（`IO32 LOW`）。
- 配置模式超时后必须自动退出并回到低功耗路径。
- 轮播间隔期间不持续高频运行（串口日志可验证状态切换）。
